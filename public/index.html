Qopy uses state-of-the-art client-side encryption:

  • 🔐 AES-256-GCM - Military-grade encryption algorithm
  • 🌐 Client-side processing - Encryption happens in your browser, not on our servers
  • 🔑 Two key strategies:
    • Random keys - For clips without passwords, we generate a random key and store it with the encrypted content
    • Password-derived keys - For password-protected clips, we derive the key from your password using PBKDF2 (100,000 iterations)
  • 🛡️ Zero-knowledge - We never see your plain text content
  • 🔄 Backward compatible - Existing unencrypted clips continue to work
  • ⚡ Automatic - No setup required, encryption happens seamlessly

How it works:

  1. You type your content in the browser
  2. Content is encrypted with AES-256-GCM before being sent to our servers
  3. Only encrypted data is stored in our database
  4. When someone retrieves the content, it's decrypted in their browser
  5. We never see the plain text at any point
